# CHANGE RULES

1. Every change must have a reason.

2. Every change must reference the affected file(s).

3. Every change must be verified before completion.

4. No undocumented changes are allowed.

5. Every architecture change must also update the relevant BWG documentation.

6. Repository history must remain traceable.
